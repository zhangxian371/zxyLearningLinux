# Interview Q&A

## 1. epoll 在项目里负责什么？

`epoll` 负责等待多个 fd 的就绪事件。服务端把监听 fd、连接 fd 和 eventfd 加入 epoll，事件到达后再分发到 accept/read/write/wakeup 处理。

## 2. 为什么使用非阻塞 fd？

因为 ET 模式下必须读写到 `EAGAIN`。如果 fd 是阻塞的，循环读写可能把 I/O 线程卡死。

## 3. EAGAIN 是错误吗？

不是。它表示当前没有更多数据可读，或当前不能继续写。连接仍然有效，应该回到事件循环等待下一次就绪。

## 4. 为什么要有输出缓冲？

`send` 不保证一次写完。短写后剩余数据必须留在连接对象的输出缓冲里，等下一次 `EPOLLOUT` 继续发送。

## 5. Reactor 解决了什么？

它把事件等待、事件分发、连接状态和业务处理拆开。这样新增业务命令时不需要改 epoll 主循环。

## 6. 为什么业务线程不能直接写 socket？

socket 和连接缓冲由 I/O 线程拥有。业务线程直接写会引入数据竞争和生命周期问题。正确做法是把结果投递回 EventLoop。

## 7. 长度前缀协议解决什么？

TCP 是字节流，不保留消息边界。长度前缀让接收方知道一条完整应用消息需要多少字节。

## 8. MySQL 和 Redis 分别负责什么？

MySQL 保存权威数据，如用户和历史消息。Redis 保存临时状态和缓存，如 token、在线状态、限流计数。

## 9. TIME_WAIT 是 bug 吗？

通常不是。主动关闭方进入 TIME_WAIT，用于处理迟到包和最后 ACK 丢失的情况。
